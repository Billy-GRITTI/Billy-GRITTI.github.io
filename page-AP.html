AP - Service DHCP sous Debian cliquer pour ouvrir

🎯 Objectifs

*Installer un serveur DHCP sous Debian
*Configurer la distribution automatique des adresses IP
*Tester la communication entre machines
*Analyser les logs Système

🖥️ Architecture réseau

Réseau utilisé : Vmnet2

⚙️ Configuration DHCP

Plage d’adresses :

192.168.1xx.50 → 192.168.1xx.80

Durée des baux :

24h (86400 secondes)

🔌 Tests réalisés

  • Ping serveur → client OK
  • Attribution IP Windows OK
  • Attribution IP Debian OK

📊 Conclusion

Le service DHCP fonctionne correctement et permet l’attribution automatique des adresses IP aux machines clientes.


AP - Installation et configuration d’un serveur HTTP (Apache) cliquer pour ouvrir

🎯 Objectifs

Créer et configurer un serveur WEB sous Linux (Apache).

🖥️ Moyens

🌐 Architecture réseau

Les machines communiquent sur le réseau VmNET3 avec une IP serveur :

192.168.xxx.180/24

📦 Installation Apache

Installation du serveur HTTP :

apt-get install apache2

⚙️ Vérification du service

  • systemctl status apache2
  • /etc/init.d/apache2 status
  • apachectl status

▶️ Gestion du service

*start → démarrer le service
*stop → arrêter le service
*restart → redémarrage complet
*graceful-stop → arrêt propre

📁 Fichiers web

Racine du site :

/var/www/html

Fichier principal :

index.html

✏️ Modification page d’accueil

Remplacement du contenu :

<h1>En fonctionnement !!! --- VotreNom --- </h1>

🌍 Test navigateur

  • Accès HTTP depuis client Debian
  • Accès HTTP depuis client Windows

🧠 Virtual Hosts

Création d’hôtes virtuels Apache :

  • a2ensite → activer un site
  • a2dissite → désactiver un site

Exemple domaine :

www.b1sisr-tzerm.net

🔎 Résumé

*Installation Apache via apt
*Configuration du site web
*Activation virtual host
*Accès depuis clients
